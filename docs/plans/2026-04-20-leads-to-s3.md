# Leads to S3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend `InstantlyToS3Workflow` so each weekly run also exports the full leads list per campaign to S3 as JSON, alongside the existing markdown analytics report.

**Architecture:** One new activity (`fetch_leads`) that POSTs to Instantly's `/api/v2/leads/list` and paginates. Existing `upload_to_s3` is generalized with an optional `content_type` parameter. The per-campaign loop inside `InstantlyToS3Workflow.run` gains a branch: if the campaign has ≥1 lead, fetch leads and upload as JSON to `campaigns/instantly-leads/{name} {date}.json`. Failure isolation remains at the per-campaign boundary.

**Tech Stack:** Python 3.11+, Temporal SDK (`temporalio`), `httpx` for HTTP, `boto3` for S3, `pydantic` models, `respx` + `moto` for tests, `pytest` (async).

**Spec:** `docs/superpowers/specs/2026-04-20-leads-to-s3-design.md`

**Environment:** Activate the project venv before any Python command:

```bash
source .venv/bin/activate
```

If `.venv` doesn't exist yet, run `python3 -m venv .venv && source .venv/bin/activate && pip install -e ".[dev]"` once.

```python
@activity.defn
async def fetch_leads(campaign_id: str) -> LeadsPayload:
    settings = get_settings()
    headers = {"Authorization": f"Bearer {settings.instantly_api_key}"}

    collected: list[dict] = []
    cursor: str | None = None

    async with httpx.AsyncClient(timeout=30.0) as client:
        while True:
            body: dict = {"campaign": campaign_id, "limit": 100}
            if cursor is not None:
                body["starting_after"] = cursor

            response = await client.post(
                INSTANTLY_LEADS_URL, headers=headers, json=body
            )
            response.raise_for_status()
            page = response.json()
            collected.extend(page.get("items", []))

            cursor = page.get("next_starting_after")
            if not cursor:
                break

    return LeadsPayload(
        leads_json=json.dumps(collected),
        lead_count=len(collected),
    )
```

```python
def build_fake_activities(state: FakeState):
    @activity.defn(name="fetch_campaigns")
    async def fetch_campaigns() -> list[CampaignData]:
        return state.campaigns

    @activity.defn(name="generate_report")
    def generate_report(campaign: CampaignData) -> ReportOutput:
        return ReportOutput(
            markdown=f"# {campaign.campaign_name}",
            campaign_id=campaign.campaign_id,
            campaign_name=campaign.campaign_name,
            metrics={},
        )

    @activity.defn(name="upload_to_s3")
    async def upload_to_s3(
        content: str, key: str, content_type: str = "text/markdown"
    ) -> None:
        if any(bad in key for bad in state.fail_upload_for):
            raise RuntimeError(f"forced failure for {key}")
        state.uploads.append((key, content, content_type))

    @activity.defn(name="fetch_leads")
    async def fetch_leads(campaign_id: str) -> LeadsPayload:
        state.fetch_leads_calls.append(campaign_id)
        if campaign_id in state.fail_fetch_leads_for:
            raise RuntimeError(f"forced fetch_leads failure for {campaign_id}")
        leads_json = state.leads_by_campaign.get(campaign_id, "[]")
        return LeadsPayload(leads_json=leads_json, lead_count=leads_json.count("{"))

    return [fetch_campaigns, generate_report, upload_to_s3, fetch_leads]
```

## Notes for the implementer

- **API shape assumption:** Instantly's pagination field names (`items`, `next_starting_after`, request body key `starting_after`) are the best-guess shape documented in the spec. If a live call reveals different field names, adjust `fetch_leads` (and the respective tests) to match — do not silently change the spec. Flag the discrepancy before merging.
- **POST vs GET:** `fetch_campaigns` uses GET. `fetch_leads` uses POST with a JSON body. Don't copy-paste the verb by accident.
- **Payload size:** The activity returns a JSON string, not a `list[dict]`, to keep the large payload crossing the workflow/activity boundary exactly once. Do not refactor `fetch_leads` to return a list.
- **Activity signature changes cascade to mocks:** When `upload_to_s3` gained the `content_type` parameter, the workflow test mock signature had to change to match. Any future signature change to registered activities needs the same treatment.
